# Лабораторная работа: Асинхронное программирование на Python с применением Flask и asyncio

**Дисциплина:** Программирование на Python  
**Технологии:** Flask, asyncio, aiohttp, Bootstrap

---
### **Задание**

#### **Теоретическая справка**

**JSONPlaceholder** – бесплатный фейковый REST API для тестирования. Предоставляет ресурсы:

- `/users` – список пользователей  
  (поля: `id`, `name`, `username`, `email`, `address`, `phone`, `website`, `company`)
- `/posts` – список постов  
  (поля: `userId`, `id`, `title`, `body`)

API поддерживает фильтрацию через query-параметры (например, `?userId=1` или `?title=qui`). Однако в рамках лабораторной работы мы будем получать полные списки и фильтровать на стороне клиента, чтобы продемонстрировать асинхронную обработку и совмещение результатов из разных источников.

**Асинхронность** позволяет выполнять несколько независимых запросов (поиск пользователей по имени, поиск постов по заголовку, поиск постов по тексту) параллельно, что значительно сокращает общее время ожидания.

#### **Задание на самостоятельную работу**

Разработать веб-приложение на Flask, которое предоставляет единую форму поиска со следующими полями (все поля необязательны для заполнения):

| Поле | Описание | API ресурс |
|------|----------|-------------|
| Имя пользователя (username) | Поиск пользователей, у которых username содержит подстроку (без учёта регистра) | `/users` |
| Email пользователя | Поиск пользователей, у которых email содержит подстроку | `/users` |
| Заголовок поста | Поиск постов, у которых title содержит подстроку | `/posts` |
| Текст поста | Поиск постов, у которых body содержит подстроку | `/posts` |

Пользователь может заполнить любое количество полей (от одного до всех четырёх). После нажатия кнопки «Найти» приложение должно:

1. **Асинхронно выполнить запросы** к соответствующим эндпоинтам JSONPlaceholder для каждого непустого поля.

   - Если заполнено поле `username` – выполняется `GET /users` и фильтрация по username.
   - Если заполнен `email` – выполняется `GET /users` и фильтрация по email (можно совместить в один запрос к пользователям, но фильтровать отдельно).
   - Если заполнен заголовок поста – `GET /posts` и фильтрация по title.
   - Если заполнен текст поста – `GET /posts` и фильтрация по body.

2. **Объединить результаты** следующим образом:

   - Пользователи, найденные по username и/или email, выводятся в отдельном блоке **«Найденные пользователи»**.
   - Посты, найденные по заголовку и/или тексту, выводятся в отдельном блоке **«Найденные посты»**.

3. **Отобразить**:

   - Для каждого пользователя: `name`, `username`, `email`, `phone`, `website`.
   - Для каждого поста: `title` (заголовок), `body` (первые 150 символов), `userId`, а также **имя автора** (должно быть получено асинхронно по `userId` через запрос к `/users/{id}`).  
     Для ускорения загрузки авторов для всех найденных постов необходимо использовать `asyncio.gather` для параллельных запросов.

---

#### **Технические требования**

- Использовать **Flask**, **aiohttp**, **asyncio**.
- Все HTTP-запросы должны быть **асинхронными**.
- Реализовать как минимум два асинхронных обработчика:
  - один для получения пользователей,
  - другой для получения постов,
  - и параллельное выполнение запросов по разным критериям.
- Если какое-либо поле не заполнено – соответствующий поиск не выполняется (пропускается).
- Обработать ситуации:
  - нет соединения,
  - пустой ответ,
  - ошибки API,
  - таймауты.
- Интерфейс оформить с помощью **Bootstrap**:
  - адаптивная форма,
  - карточки результатов,
  - вкладки или отдельные блоки для пользователей и постов.
- Добавить **комментарии к коду**, поясняющие асинхронные конструкции и логику параллельного выполнения.
